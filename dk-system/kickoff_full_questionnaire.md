為什麼做
1. 不做這件事會發生什麼具體後果？（若可，請量化）
2. 此專案與本季／本年部門優先目標有何關聯？會改善哪些 KPI？
3. 為什麼要現在做，而不是六個月後？有時間敏感因素嗎？
4. 我們有哪些證據支持這個方向是對的？

做什麼
5. 結束時的交付物為何？請列出 MVP 與後續可選項目。
6. 本階段不做哪些項目（邊界）？
7. 功能優先順序為何？（Phase 1 / 2 / 3）
8. 依賴清單：需要哪些 teams / systems / API / 權限？誰負責提供？

成功定義
9. 主要與次要量化指標為何？（例如每週節省 X 小時、採用率 ≥ Y%）
10. 工程師採用的必要條件是什麼？（格式一致性、可靠性、可回退流程等）
11. 預期時程：PoC / Beta / 正式上線的目標時間為何？

風險與歷史
12. 你認為目前最大的 1–3 個風險是什麼？（若有，請列出緩解方向）
13. 過去是否做過類似系統？現狀如何？是否有可重用的程式碼或文件？請附位置
14. 供應商或外部來源的規格是否會頻繁變更？過去變動頻率如何？
15. 長期維運由誰負責？若非同一團隊，維運分工如何安排？
