# Step 1：Kickoff 訪談引導

## 引導方式
- 一次只問一個面向
- 口語化，不要照本宣科
- 回答後確認有沒有需要追問的，再進入下一個面向
- 答案模糊時用具體化技巧

## 四層問題

### 第一層：為什麼做
→ 「不做的話會發生什麼？」
→ 「跟今年優先目標有什麼關係？」
→ 「為什麼現在做，不是六個月後？」

### 第二層：做什麼
→ 「結束時要交付什麼？」
→ 「什麼是不做的？邊界在哪？」
→ 「如果有爭議，最後誰說了算？」
→ 「這件事做起來，哪些其他 team 會被影響到？」
→ 「這個設計方向是怎麼決定的？有沒有考慮過其他方式？」

### 第三層：成功定義
**Business Value：**
→ 「這個專案做完，對業務最直接的影響是什麼？」
→ 「有沒有可以量測的數字？比如省多少時間、提升多少效率？」

**Justification：**
→ 「我們怎麼知道這個方向是對的？有沒有驗證過？」
→ 「stakeholder 最在意的是什麼？」

### 第四層：風險在哪
→ 「這個專案能成功，最關鍵的前提是什麼？」
→ 「哪個地方你最擔心出問題？」
→ 「以前有沒有人試過類似的東西？如果有，為什麼沒有成功？」
→ 「有哪些事情不是我們能決定，但會影響結果的？」
→ 「以前有沒有類似的系統存在？現在還在用嗎？」

## 具體化技巧（答案模糊時用）
| 情況 | 說法 |
|---|---|
| 答案很模糊 | 「可以舉一個具體的例子嗎？」（舉例法）|
| 說不清楚差異 | 「做完之後跟現在最大的不同會是什麼？」（對比法）|
| 無法量化 | 「有沒有辦法變成一個數字？」（量化法）|
| 三個都試了還是模糊 | 「我現在理解的是 XXX，你覺得這樣說對嗎？」|
